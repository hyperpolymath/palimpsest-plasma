= Palimpsest Plasma — Contractiles (Delegated)
:toc:
:sectnums:

== Overview

This directory delegates operational contractile execution to the canonical
**contractile CLI** from `reposystem/contractiles/` rather than reimplementing
the contractile stack locally.

PLASMA's governance-layer contractile _concept_ (tension, holding, relaxing,
releasing — see `docs/contractiles.adoc`) remains PLASMA-specific. But the
operational files (Mustfile, Dustfile, Intentfile, K9 templates) are consumed
from the shared contractile CLI, not maintained as local copies.

== Architecture

[source]
----
palimpsest-plasma/contractiles/
  README.adoc          ← This file (delegation rationale)
  contractile.just     ← Imports contractile CLI recipes

reposystem/contractiles/
  cli/                 ← Canonical must/trust/dust/intend/k9 binaries
  mustfile/            ← Mustfile format spec + Ada implementation
  trustfile/           ← Trustfile definitions
  dustfile/            ← Dustfile definitions
  ...
----

== How It Works

1. **`contractile.just`** imports the contractile CLI recipes via Just's
   `import` mechanism (see root `Justfile`).
2. **`must check`**, **`trust verify`**, **`dust status`**, **`intend list`**,
   and **`k9 eval`** all delegate to the shared contractile CLI binaries.
3. PLASMA's governance engine (`src/core/plasma_governance.ml`) consumes
   contractile _findings_ as typed inputs, applying its own tension/relaxation
   model on top.

== Relationship to PLASMA Governance Contractiles

The formal PLASMA contractile model (typed OCaml structures with
`Tightening | Holding | Relaxing | Released` states) is documented in
`docs/contractiles.adoc`. That model _wraps_ the operational contractile CLI
outputs:

* `must check` findings become facts in PLASMA's evaluation engine
* `trust verify` results feed provenance contractiles
* `dust status` informs reversibility contractiles
* `intend list` feeds intent/roadmap contractiles
* `k9 eval` validates K9-SVC components

The PLASMA engine adds governance semantics (tension scores, exhibit exemptions,
Council decisions) on top of these raw contractile results.

== Migration Notes

Previously, this directory contained local copies of Mustfile, Dustfile,
Intentfile, and K9 templates. These have been removed in favour of consuming
the contractile CLI from `reposystem/contractiles/`. The templates are available
in `rsr-template-repo/docs/templates/contractiles/` for bootstrapping new repos.
