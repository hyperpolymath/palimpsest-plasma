== CLI Interface

One binary (`plasma`), narratively clear subcommands, machine-readable
output where useful.

=== Implemented

[source]
----
plasma check [PATH] [--policy FILE] [--format human|json|sarif]
             [--severity error|warning|info] [--quiet] [--verbose]
plasma fix [PATH] [--policy FILE] [--license SPDX-ID] [--author NAME]
           [--apply] [--no-backup] [--format human|json]     # v0.3
plasma facts [PATH] [--with-contents]
plasma diff BEFORE.json AFTER.json [--format human|json]     # v0.4
plasma policy validate FILE
plasma audit [PATH] [--license SPDX-EXPR] [--fix]
plasma init [--path PATH] [--license SPDX-ID] [--author NAME]
plasma badge [--license SPDX-ID] [--style STYLE]
plasma migrate [--path PATH] [--from SPDX-ID] [--to SPDX-ID]
----

=== `plasma check`

Evaluate a repository against a policy.

* **Input:** repo path (default `.`), policy file (default: the bundled
  repo-hygiene policy)
* **Output:** exit 0 when no violation at or above the severity threshold
  exists; exit 1 otherwise. Human text, JSON (the full `Evaluation`), or
  SARIF 2.1.0.
* **Flags:**
  - `--policy FILE` — custom policy (`.toml` or `.json`)
  - `--format human|json|sarif` (default: human)
  - `--severity error|warning|info` — failure threshold (default: error)
  - `--quiet` — exit code only
  - `--verbose` — include satisfied (pass) findings

=== `plasma fix`

Plan or apply corrective actions for policy violations. **Dry-run by
default** — it prints the plan and touches nothing; `--apply` makes the
changes, writing a `<file>.bak` backup before modifying each file (unless
`--no-backup`).

The planner is pure (`plasma_engine::plan`); applying is the IO boundary
(`plasma_engine::apply`). Every violation is either turned into a
mechanical action or listed under *Manual* with a reason — nothing is
silently unfixable. The auto-action set is:

* **AddSpdxHeader** — a source file that must carry an SPDX header but does
  not gets one prepended, using `--license` (default MPL-2.0), `--author`,
  and the correct comment prefix for the file type.
* **CreateFile** (v0.5) — a required file at a *concrete* path (not a glob)
  that is missing is created with obvious placeholder content. Never
  overwrites an existing file.

A required file named by a *glob* (e.g. `LICENSE*` — no single filename to
create) and unparsable headers are reported as manual. Applying is
idempotent: a file that already has a header, or already exists, is skipped.

* **Flags:** `--policy FILE`, `--license SPDX-ID`, `--author NAME`,
  `--apply`, `--no-backup`, `--format human|json`
* **Exit:** dry-run returns 1 when actions are pending (so CI can gate on a
  clean tree), 0 when there is nothing to do; `--apply` returns 0 unless an
  action errored.

Relationship to `plasma audit --fix`: `audit --fix` is the license-specific
tool (zone-expected headers from `.plasma.toml`); `plasma fix` is the
general policy-remediation surface driven by engine findings — headers are
simply its first supported action class.

=== `plasma facts`

Dump the deterministic fact snapshot the evaluator sees, as JSON. This is
the diffable before/after artifact for agent-verification tooling.

* `--with-contents` — also include text-file contents (`file_contents`),
  needed for content-aware policies. Off by default so the snapshot stays
  lean; `plasma check`/`fix` enable it automatically when the policy uses a
  `file-matches-pattern` condition.

=== `plasma diff`

Compare two fact snapshots produced by `plasma facts`. This is the
verification protocol for agent claims: snapshot before the agent runs,
snapshot after, diff the two — the result is exactly what changed as far
as policy evaluation is concerned (files added/removed, SPDX header
transitions, metadata changes, git state).

* **Exit semantics follow `diff(1)`:** 0 = identical, 1 = differ,
  2 = error — scriptable without parsing output.
* Output is deterministic: identical snapshot pairs give byte-identical
  JSON. The format is pinned in
  link:interchange-contracts.adoc[interchange-contracts.adoc].

=== `plasma policy validate`

Parse and schema-check a policy file; print rule counts on success. Fails
with a precise error on schema-version mismatch, duplicate rule ids, or
reserved constructs.

=== `plasma audit`

Zone-aware SPDX header audit powered by plasma-parser. Reads `.plasma.toml`
when present; otherwise expects `--license` (default MPL-2.0) everywhere.
`--fix` inserts missing headers using each file's zone-expected license.
Exits non-zero on non-compliance so CI fails.

=== `plasma init` / `migrate` / `badge`

License adoption for any SPDX identifier (MPL-2.0 default). `init` embeds
the full MPL-2.0 text and writes stubs plus a warning for other licenses;
`migrate` rewrites LICENSE (with backup) and sweeps SPDX headers from the
detected or given `--from` to `--to`; `badge` emits shields.io markup with
proper `-`→`--` escaping.

== Roadmap surfaces (not yet implemented)

* More `plasma fix` actions — `UpdateField` (beyond the v0.3 `AddSpdxHeader`
  and v0.5 `CreateFile` actions)
* Git hooks (pre-commit/pre-push wrapping `plasma check --quiet`) and a
  reference GitHub Action / GitLab job (v0.4)
* `plasma governance` / decision registries — awaits `consistent-with`
  semantics
* Daemon/cron mode for periodic audits

A future `.plasma.toml` extension will let a repository pin its own policy
file so `plasma check` needs no flags at all.

== Exit codes

[cols="1,3"]
|===
| Code | Meaning

| 0 | Clean: no violation at or above the threshold
| 1 | Violations at or above the threshold (check), or non-compliant files (audit)
| 2 | Usage or IO error (bad flags, unreadable policy, missing path)
|===

== Output Formats

* **Human** — violations with rule id, subject, message, rationale, and
  source; a one-line summary. Pass findings shown with `--verbose`.
* **JSON** — the `Evaluation` structure serialized directly; deterministic
  (byte-identical for identical inputs).
* **SARIF 2.1.0** — `plasma/<rule-id>` rule namespace, levels
  `error|warning|note`, pass findings as `kind: "pass"` under `--verbose`.
  Uploadable to GitHub Code Scanning.
